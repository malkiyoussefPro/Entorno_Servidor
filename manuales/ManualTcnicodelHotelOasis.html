4. Requisitos de Hardware y Software

   4.1 Especificaciones mínimas del servidor.

Para proporcionar un conjunto básico de especificaciones mínimas del servidor, considere lo siguiente:

  1. Procesador (CPU):  Se recomienda al menos un procesador de doble núcleo con una velocidad de reloj de al menos 2.0 GHz. Para sitios web más grandes o con mayor carga de trabajo, se pueden necesitar procesadores más potentes o con más núcleos.
  2. Memoria RAM: Se recomienda al menos 4 GB de RAM para un servidor web básico. Para sitios web más complejos con mayor tráfico o aplicaciones que requieren más recursos, se pueden necesitar cantidades mayores de RAM.
  3. Almacenamiento: Se recomienda un disco duro de al menos 100 GB de almacenamiento. Esto proporciona suficiente espacio para el sistema operativo, el software del servidor web, los archivos del sitio web y cualquier base de datos asociada. También se puede considerar el uso de unidades de estado sólido (SSD) para un mejor rendimiento de E/S.
  4. Ancho de Banda: El ancho de banda necesario dependerá del tráfico esperado del sitio web y del tamaño de los archivos que se descarguen. Se debe asegurar un ancho de banda adecuado para manejar el tráfico entrante y saliente sin problemas.
  5. Sistema Operativo: Se puede utilizar un sistema operativo basado en Linux, como Ubuntu Server, CentOS o Debian. Estos sistemas operativos son comunes para servidores web debido a su estabilidad, seguridad y disponibilidad de software de servidor, se puede utilizar Window a partir de la versión 7
  6. Software del Servidor Web: Se puede utilizar Apache XAMPP HTTP Server o Nginx como servidor web principal. Ambos son populares y ampliamente utilizados en la industria

Estas son solo especificaciones mínimas y pueden variar dependiendo de las necesidades específicas del sitio web y del entorno de producción. Es importante evaluar cuidadosamente los requisitos de hardware y software para garantizar un rendimiento óptimo y una alta disponibilidad del sitio web.

5. Diseño y Experiencia del Usuario

   5.1 Principios de diseño utilizados.

Al diseñar un sitio web, es fundamental seguir principios de diseño que mejoren la experiencia del usuario y la estética del sitio.

  1. Usabilidad:  El sitio web debe ser fácil de usar y navegar. Por lo tanto he usado los elementos de diseño organizados de manera lógica y clara, con una navegación intuitiva que permita a los usuarios encontrar la información que buscan de manera rápida y sencilla.
  2. Accesibilidad: El diseño del sitio web debe ser accesible para todos los usuarios. Esto implica utilizar técnicas de contraste adecuado entre texto y fondo, y estructuras de página semánticas que funcionen bien y que se interactúen entre sí .
  3. Diseño Responsivo: El sitio web debe ser compatible con una variedad de dispositivos y tamaños de pantalla. El diseño responsivo garantiza que el sitio se vea bien y funcione correctamente en dispositivos móviles, tabletas y computadoras de escritorio.
  4. Simplicidad:  Un diseño limpio y simple minimalista ayuda a reducir la confusión, al  simplificar la interfaz puede mejorar la usabilidad y la estética del sitio.
  5. Consistencia: El diseño del sitio web debe ser coherente en todas sus páginas. Esto incluye el uso consistente de colores, fuentes, estilos de botones y elementos de navegación en todo el sitio para crear una experiencia unificada para el usuario, en la página Hotel Oasis he usado unos colores que no molestan a nivel visual y una fuente agradable para lograr una experiencia inolvidable  .

8. Seguridad:

   8.1 Medidas de seguridad implementadas ( HTTPS, autenticación de usuarios).

  • HTTPS (Protocolo seguro de transferencia de hipertexto):

El sitio web utiliza HTTPS en lugar de HTTP para asegurar la comunicación entre el navegador del usuario y el servidor. HTTPS cifra los datos transmitidos, lo que protege la privacidad y la integridad de la información, especialmente durante la transferencia de datos sensibles como contraseñas y datos de pago.

  • Autenticación de Usuarios:
  • Se implementa un sistema de autenticación de usuarios para garantizar que solo usuarios autorizados puedan acceder a ciertas áreas del sitio web, como perfiles de usuario o información privada. Esto implica el uso de nombres de usuario y contraseñas únicas para cada usuario por ejemplo los clientes no pueden acceder al dashboard
